# Security Policy

## Reporting a Vulnerability

Please report suspected vulnerabilities privately instead of opening a public issue.

Email: security@evolink.ai

Include the affected repository, a concise impact summary, reproduction steps, and any relevant logs or request IDs. Do not include API keys, secrets, or private user data in screenshots or issue comments.

## Response Expectations

We triage security reports as soon as possible and prioritize fixes by user impact, exploitability, and affected surface area. If a report affects an EvoLink API integration, include the model or endpoint name so we can route it to the right owner.

## Scope

This policy applies to public repositories under the `Evolink-AI` organization unless a repository provides a more specific `SECURITY.md`.
